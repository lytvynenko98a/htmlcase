Умови

Базові поняття. Що таке "умовні оператори"? "if"?

Умовні оператори: if, '?'

Іноді, залежно від умови, потрібно виконувати різні дії. Для цього використовується оператор if.

Наприклад:

var year = prompt ('У якому році з'явилася специфікація ECMA-262 5.1 ?', ' ');

if (year != 2011) alert ('А ось і неправильно!');

Оператор if

Оператор if («якщо») отримує умову, у прикладі вище це year != 2011. Він обчисює її, і якщо результат - true, то виконує команду.

Якщо потрібно виконати більше однієї команди - вони оформляються блоком коду в фігурних дужках:

if (year ! = 2011) {

alert ( 'А ось ..' );

alert ('..і неправильно!' );

}

Рекомендується використовувати фігурні дужки завжди, навіть коли команда одна.

Це покращує читабельність коду.

Перетворення до логічного типу

Оператор if (...) обчислює і перетворює вираз в дужках до логічного типу.

У логічному контексті:

- Число 0, порожній рядок "", null і undefined, а також NaN є false,

- Решта значень - true.

Наприклад, така умова ніколи не виконається:

if (0) { // 0 перетвориться до false ...

}

... А така - виконається завжди:

if (1) { // 1 перетвориться до true ...

}

Можна і просто передати вже готове логічне значення, наприклад, заздалегідь обчислене в змінній:

var cond = (year != 2011); // True / false

if (cond) {

...

}

Невірна умова, else

Необов'язковий блок else («інакше») виконується, якщо умова невірна:

var year = prompt ( 'Введіть рік, у якому зявився стандарт ECMA-262 5.1', ​​' ' );

if (year == 2011) {

alert ( 'Та ви знавець!' );

} еlse {

alert ( 'А ось і неправильно!' ); // Будь-яке значення, крім 2011

}

Декілька умов, else if

Буває потрібно перевірити кілька варіантів умови. Для цього використовується блок else if .... Наприклад:

var year = prompt ( 'У якому році з'явилася специфікація ECMA-262 5.1?', ' ' );

if (year < 2011) {

alert ( 'Це занадто рано ..' );

} else if (year > 2011) {

alert ( 'Це запізно ..' );

} else {

alert ( 'Так, справді в цьому році!');

}

У прикладі вище JavaScript спочатку перевірить першу умову, якщо вона хибна - перейде до другої - і так далі, до останнього else.

Оператор знак питання "?"

Іноді потрібно залежно від умови привласнити змінну. Наприклад:

var access;

var age = prompt ( 'Скільки вам років?', ' ' );

if (age > 14) {

access = true;

} еlse {

access = false;

}

alert (access);

Оператор знак питання '?' дозволяє робити це коротше і простіше.

Він складається з трьох частин:

умова ? значення1 : значення2

Перевіряється умова, потім коли вона вірна - повертається значення1, якщо невірна - значення2, наприклад:

access = (age > 14) ? true : false;

Оператор '?' виконується після більшості інших, зокрема - пізніше порівнянь, тому дужки можна не ставити:

access = age > 14 ? true : false;

... Але коли дужки є - код краще читається. Так що рекомендується їх писати.

На замітку:

В даному випадку можна було б обійтися і без оператора '?', бо порівняння само по собі вже повертає true / false:

access = age > 14;

«Тернарний оператор»

Знак питання - єдиний оператор, у якого є аж три аргументи, у той час як у звичайних операторів їх один-два. Тому його називають «тернарний оператор».

Кілька операторів "?"

Послідовність операторів ' ? ' дозволяє повернути значення в залежності не від однієї умови, а від декількох.

Наприклад:

var age = prompt ( 'вік?' , 18);

var message = (age < 3) ? 'Здрастуй, малюк!' : (age < 18) ? 'Вітання!' : (age < 100)? 'Вітаю!' : 'Який незвичайний вік!';

alert (message);

Спочатку може бути складно зрозуміти, що відбувається. Однак, уважно придивившись, ми помічаємо, що це звичайна послідовна перевірка!

Знак питання перевіряє спочатку age < 3, якщо вірно - повертає 'Здрастуй, малюк!', Якщо ні - йде за "дві крапки" і перевіряє age < 18. Якщо це вірно - повертає 'Привіт!', інакше перевірка age < 100 і 'Вітаю! '... І нарешті, якщо нічого з цього не вірно, то ' Який незвичайний вік! '.

Те ж саме через if..else:

if (age < 3) {

message = 'Здрастуй, малюк!';

} else if (age < 18) {

message = 'Привіт!';

} else if (age < 100) {

message = 'Вітаю!';

} else {

message = 'Який незвичайний вік!';

}

Нетрадиційне використання "?"

Іноді оператор знак питання ' ? ' використовують як заміну if:

var company = prompt ( 'Яка компанія створила JavaScript?', ' ');

(company == 'Netscape') ?

alert( 'Так, вірно' ): alert( 'Неправильно' );

Працює це так: залежно від умови, буде виконана або перша, або друга частина після '?'.

Результат виконання не присвоюють в змінну, так що він пропаде (втім, alert нічого не повертає).

Рекомендується не використовувати знак питання таким чином.

Незважаючи на те, що з вигляду такий запис коротше if, він є менш читабельним.

Ось, для порівняння, те ж саме з if:

var company = prompt ('Яка компанія створила JavaScript?', '');

if (company == 'Netscape') {

alert ( 'Так, вірно' );

else {

alert ('Неправильно');

}

При читанні коду око йде вертикально і конструкції, що займають кілька рядків, зі зрозумілою вкладеністю, сприймаються набагато легше. Можливо, ви й самі відчуєте, пробігшись очима, що синтаксис з if більш простий і очевидний ніж з оператором '?'.

Сенс оператора '?' - Повернути те чи інше значення, залежно від умови. Будь ласка, використовуйте його за призначенням, а для виконання різних гілок коду є if.
