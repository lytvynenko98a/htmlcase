Форми

Що таке форма? Як зробити просту форму?

Найпростіша форма

<!DOCTYPE html>

<html>

<head>

<meta charset="utf-8">

<title>Форма для входу на сайт</title>

</head>

<body>

<h1>Форма входу</h1>

<form action="http://man.adr.com.ua/html/form" method="post">

<label for="login-field">Ваш логін</label>

<input type="text" name="login" id="login-field">

<br><br>

<label for="password-field">Ваш пароль</label>

<input type="password" name="password" id="password-field">

<br><br>

<input type="submit" value="Ввійти">

<br><br>

</form>

</body>

</html>

1.Створюємо форму. Тег <form>

Форми потрібні для того, щоб відправляти дані з веб-сторінки на веб-сервер, який зможе ці дані обробити: зареєструвати користувача, створити повідомлення на форумі, відправити листа і так далі. Загалом, форми в вебі просто необхідні.

Щоб створити форму, потрібно використовувати парний тег <form>, усередині якого розміщуються поля форми.

    У тега <form> є два важливі атрибути:

  • "action" - задає адресу, URL, відправка форми;

  • "method" - задає метод надсилання форми.

Приклад:

<form action = "http://http://man.adr.com.ua/form" method = "get">

поля форми

</form>

Методи

Для відправки форми зазвичай використовують методи "get" або "post". Якщо не вказати атрибут "method", то буде використаний "get".

Метод "get" посилає дані форми в рядку запиту, тобто вони видні в адресному рядку браузера і слідують після знака питання.

Наприклад:

https://www.google.com/search?q=man

Метод "get" краще використовувати в пошукових формах, тому що він дозволяє отримати посилання на результати пошуку і передати її комусь.

Метод "post" посилає дані в тілі HTTP-запиту і використовується, коли потрібно відправити багато даних і посилання на результат обробки цих даних не потрібні. Наприклад, при редагуванні особистого профілю.

2.Текстове поле введення <input>

Більшість полів форм створюється за допомогою одиночного тега <input>.

    У цього тега два обов'язкові атрибути:
  • "type" - задає тип поля;
  • "name" - задає ім'я поля.

Тип поля впливає на те, як воно буде відображатися і поводитися. Найпоширеніший тип - це "text", який позначає текстове поле. Він же використовується за замовчуванням.

Приклад:

<form action = "http://http://man.adr.com.ua/form" method = "get">

<input type="text" name="login">

</form>

Ім'я поля потрібно, щоб правильно обробити дані на сервері. Зазвичай, ім'я поля має бути унікальним в межах форми, хоча є винятки. Для завдання імені поля використовують латинські букви і цифри.

3.Ідентифікатор "id" і значення за замовчуванням

Атрибут "id" поля введення позначає ідентифікатор. Він повинен бути унікальним не тільки в межах форми, а й на всій сторінці.

Зазвичай ідентифікатори використовують для підвищення зручності роботи з формою, наприклад, створюють підписи, пов'язані з дрібними полями. При клацанні по таких підписах активується зв'язане поле. І це зручно, так як по великому підпису потрапити легше, ніж по маленькому полю. Також ідентифікатори використовують в JavaScript для роботи з полями.

Ідентифікатор на відміну від імені поля не передається на сервер. Краще використовувати ідентифікатори, що відрізняються від імені поля, особливо актуально це для полів множинного вибору, які ми розберемо далі.

Атрибут "value" задає значення поля вводу за замовчуванням. Це теж підвищує зручність.

Погодьтеся, приємно зайти у величезну анкету на яких-небудь держпослуг, а там ваші паспортні дані вже підставлені в потрібні поля і заповнювати їх не треба. І все завдяки тому, що програміст додав до полів атрибут value з потрібними даними.

<input type="password" name="password" id="password-field">

Кількість <input type="text" name="kilkist" id="kilkist-field" value="507"> шт.

4.Підпис для поля вводу

Можна подумати, що зробити підпис до поля дуже просто.

Пишемо текст поряд з полем і все готово:

Підпис <input type = "text" name = "username">

Насправді цього недостатньо - ми отримали просто шматок тексту і поле, які розташовані поруч один з одним, але логічно ніяк не пов'язані.

Є спеціальний тег, який дозволяє сміливо сказати: «Цей шматок тексту дійсно підпис до цього поля!». Це парний тег <label>.

Він пов'язує текст і поле введення логічно.

Створювати підписи до полів за допомогою <label> - хороший прийом. Використовуйте його.

Перший спосіб створити підпис - просто обернути текст підпису та тег поля в тег <label>, ось так:

<label>

Підпис <input type = "text" name = "username">

</label>

Треба відзначити, що при обертанні тексту в тег <label> він візуально ніяк не змінюється, адже головне завдання підпису - створити логічний зв'язок.

5. Пов'язуємо підпис і поле по "id"

Іноді обернути поле і текст підпису в тег <label> не можна. Наприклад, коли вони розміщені в різних комірках таблиці.

У цьому випадку можна зв'язати підпис з полем за допомогою атрибуту "id". Алгоритм такий:

Додаємо до поля введення ідентифікатор за допомогою атрибуту "id".

Обертаємо текст підпису в тег <label>.

Додаємо тегу <label> атрибут "for".

У атрибут "for" записуємо таке ж значення, що і в атрибуті "id" у поля.

наприклад:

<label for="login-field">Ваш логін</label>

<input type="text" name="login" id="login-field">

6. Поле для введення пароля

Ми створюємо просту форму входу. Вже знаємо, як створити два поля і пов'язані з ними підписи. Одне з полів призначено для введення пароля, але зараз воно є простим текстовим полем.

<label for="login-field">Ваш логін</label>

<input type="text" name="login" id="login-field">

<br><br>

<label for="password-field">Ваш пароль</label>

<input type="text" name="password" id="password-field">

<br><br>

Щоб зробити його справжнім полем для введення пароля, в якому текст буде відображатися «зірочками», потрібно просто змінити значення атрибута "type" на "password".

<label for="login-field">Ваш логін</label>

<input type="text" name="login" id="login-field">

<br><br>

<label for="password-field">Ваш пароль</label>

<input type="password" name="password" id="password-field">

<br><br>

7. Кнопка відправки форми

Форма практично готова. Залишилося додати кнопку для відправки форми. Така кнопка створюється за допомогою тега <input> c типом "submit".

Напис на кнопці можна задати за допомогою атрибуту "value". Для кнопки відправки форми задавати ім'я необов'язково. Але якщо ім'я задано, то на сервер будуть відправлятися ім'я і значення кнопки.

Зазвичай ім'я для кнопки відправки задають, коли у формі кілька кнопок, що відповідають за різні дії. Браузер відправляє на сервер ім'я і значення тільки тієї з них, на яку натиснув користувач. Таким чином, сервер може зрозуміти, яку кнопку натиснули і що потрібно зробити.

<input type="submit" value="Ввійти">

Наша перша форма готова.

8.Багаторядкові поля введення <textarea>

Ми навчилися створювати найпростіші форми з текстовими полями і кнопками. А тепер познайомимося з більш складними елементами форми.

Багаторядкове текстове поле створюється за допомогою парного тега <textarea>. У нього є атрибути "name" і "id", які аналогічні атрибутам текстового поля.

Атрибут "rows" приймає цілочисельне значення і задає висоту багаторядкового поля в рядках. Атрибут cols задає ширину поля в символах. В якості ширини символу береться деяка «середня ширина».

Атрибут "value" у багаторядкового поля відсутній, а значення за замовчуванням задається по-іншому. Текст, розташований між відкритим і закритим тегом <textarea> і є значенням за замовчуванням. Ось так:

<textarea> Значення за замовчуванням </textarea>

Приклад (форма2):

<!DOCTYPE html>

<html>

<head>

<meta charset="utf-8">

<title>Багаторядкове поле вводу</title>

</head>

<body>

<h1>Коментарі</h1>

<form action="http://man.adr.com.ua/html/form" method="post">

<p>Список коментарів</p>

<p>Введіть ваш коментар:</p>

<textarea name="comment" id="comment-field" rows="10">Текст коментаря</textarea>

<p><input type="submit" value="Відправити"></p>

</form>

</body>

</html>

9.Чекбокс або «галочка»

Поле-галочка - це тег <input> з типом "checkbox".

Галочка працює за принципом «або так, або ні». Якщо галочка стоїть, то браузер посилає змінну з ім'ям поля на сервер, якщо галочки немає, то не надсилається нічого. Таким чином, атрибут "value" не є обов'язковим.

Щоб галочка стояла за замовчуванням, потрібно додати до тегу атрибут "checked". Ось так:

<input type = "checkbox" checked>

Якщо в одній формі є кілька чекбоксів, то імена у них повинні бути різними.

<p>Запитання:</p>

<input name="subscribe" type="checkbox" id="subscribe-field" checked>

<label for="subscribe-field">Підписати вас на россилку?</label>

Приклад (форма3):

<!DOCTYPE html>

<html>

<head>

<meta charset="utf-8">

<title>Форма реєстрації</title>

</head>

<body>

<h1>Реєстрація</h1>

<form action="http://man.adr.com.ua/html/form" method="post">

<label for="login-field">Ваш логін (email):</label><br>

<input type="text" name="login" id="login-field">

<br>

<label for="password-field">Пароль:</label><br>

<input type="password" name="password" id="password-field">

<br>

<label for="comment-field">Інформація про себе:</label><br>

<textarea name="comment" id="comment-field" rows="3"></textarea>

<br>

<input name="subscribe" type="checkbox" id="subscribe-field" checked>

<label for="subscribe-field">Підписатись на розсилку</label><br>

<input type="submit" value="Зареєструватись">

</form>

</body>

</html>

10. Перемикач або «радіобаттон»

Поле-перемикач - це тег <input> з типом "radio".

Зазвичай перемикачі розміщують групами по кілька штук. Причому у перемикачів з однієї групи має бути однакове ім'я і різні значення, які задаються c допомогою "value".

Таким чином, атрибут "value" є для перемикачів обов'язковим. Браузер відправляє на сервер значення "value" вибраного перемикача.

У цьому завданні ми почнемо створювати перемикач.

<h1>Тест</h1>

<form action=index.html method="get">

<p><i>Який тег позначає «абзац»?</i></p>

<label>

<input type="radio" name="question-one" value="p">

Тег <p>

</label>

11. Група перемикачів

Тепер додамо ще один варіант відповіді в наш перемикач. Для цього потрібно додати ще один <input> з таким же ім'ям, але іншим значенням "value".

Подібним чином можна створювати групи перемикачів з будь-якою кількістю варіантів.

Щоб зробити який-небудь варіант в перемикачі обраним за замовчуванням, потрібно додати до відповідного тегу <input> атрибут "checked", як у поля-галочки.

До речі, ім'я поля у перемикачів однієї групи має бути однаковим, але ідентифікатори завжди повинні бути унікальними.

Приклад (форма 4):

<!DOCTYPE html>

<html>

<head>

<meta charset="utf-8">

<title>Группа перемикачів</title>

</head>

<body>

<h1>Тест</h1>

<form action="index.html" method="get">

<p>В тестах часто потрібно вибрати один з варіантів</p>

<p><i>Який тег позначає «абзац»?</i></p>

<label>

<input type="radio" name="question-one" value="p">

Тег <p>

</label>

<br>

<label>

<input type="radio" name="question-one" value="b" checked>

Тег <b>

</label>

<input type="submit" value="Відправити">

</form>

</body>

</html>

12. Список, що розкривається або «селект»

Список, що розкривається також, як і перемикач, дозволяє вибрати один варіант відповіді з декількох.

Список, що розкривається створюється за допомогою парного тега <select>, у якого є знайомі атрибути "name" і "id".

Варіанти відповідей задаються за допомогою парних тегів <option>, які повинні розташовуватися всередині тега <select>.

Наприклад:

<select name = "країни">

<option value = "Україна">Україна </option>

<option value = "США">США</option>

...

</select>

В атрибуті "value" тега <option> задається значення варіанта відповіді, а всередині цього тега розташовується підпис варіанту відповіді.

Якщо при відправленні форми у обраного варіанту заданий "value", то на сервер відправиться значення цього атрибута. В іншому випадку буде відправлений текст підпису.

Приклад (форма 5):

<!DOCTYPE html>

<html>

<head>

<meta charset="utf-8">

<title>Список «селект»</title>

</head>

<body>

<h1>Анкета</h1>

<form action="index.html" method="get">

<p>вибираємо дату</p>

<p><i>Ваш рік народження</i></p>

<select name="year">

<option value="2000">2000 рік</option>

<option value="2001">2001 рік</option>

...

</select>

<input type="submit" value="Відправити">

</form>

</body>

</html>

13. «Мультіселект»

Список, що розкривається можна перетворити на так званий «мультіселект», тобто список, в якому можна вибрати не один, а кілька варіантів.

Щоб зробити це, потрібно додати до тегу <select> атрибут "multiple".

Вибрати декілька варіантів можна, клацаючи по них з натисненою клавішею "Ctrl" на "Windows".

Висоту мультіселекта можна змінювати за допомогою атрибуту "size" тега <select>.

Щоб відзначити як вибрані за замовчуванням одне або кілька значень, потрібно до відповідних тегів <option> додати атрибут "selected".

Приклад (форма 6):

<!DOCTYPE html>

<html>

<head>

<meta charset="utf-8">

<title>«Мультиселект»</title>

</head>

<body>

<h1>Анкета</h1>

<form action="index.html" method="get">

<p><i>Улюблені дні тижня:</i></p>

<select name="days" size="7" multiple>

<option>Понеділок</option>

<option>Вівторок</option>

<option>Середа</option>

<option>Четвер</option>

<option>П'ятниця</option>

<option>Субота</option>

<option selected>Неділя</option>

</select>

<input type="submit" value="Вибрати">

</form>

</body>

</html>

14. Поле для завантаження файлів

Поле для завантаження файлів - це тег <input> з типом "file". Для цього поля обов'язковим атрибутом є ім'я.

Щоб поле запрацювало і браузер зміг передати вибраний файл на сервер, необхідно додати формі атрибут "enctype" зі значенням "multipart / form-data". Не полю, а формі.

Приклад (форма 7):

<!DOCTYPE html>

<html>

<head>

<meta charset="utf-8">

<title>Поле для завантаження файлів</title>

</head>

<body>

<h1>Профіль</h1>

<form action="http://man.adr.com.ua/html/form" method="post" enctype="multipart/form-data">

<p>Виберіть зображення для свого аватара:</p>

<input type="file" name="avatar">

<input type="submit" value="Завантажити">

</form>

</body>

</html>

15. Приховане поле

І ще одне невидиме і дуже корисне поле. Це приховане поле. Його використовують, коли у формі потрібно відправити якісь додаткові службові дані, які не вводяться користувачем.

Наприклад, це можуть бути реквізити замовлення або номер користувача у формі оплати.

Приховане поле - це тег <input> з типом "hidden".

<label>

Сумма:

<input type="text" name="amount" value="500">грн.

</label><br><br>

<label>

<input type="hidden" name="account_number" value="15">

</label>

<input type="submit" value="Заплатити">
